# 履歷解析規則

## 萃取優先順序

1. **明確標示的欄位**（如「技能」、「Skills」區塊）優先
2. **從工作經歷推斷**技能（例如：描述中提到「使用 Python 開發...」→ 加入 Python）
3. **不確定的欄位**留空，不要猜測

## 技能分類

萃取技能時，請區分：
- `hard_skills`：程式語言、工具、框架、證照等可量化技能
- `soft_skills`：溝通、領導、專案管理等軟實力（從描述中推斷）

## 年資計算

- 計算所有工作經歷的總月數，轉換為年（四捨五入到 0.5 年）
- 若有重疊期間（如兼職），以最長連續時間計算，不重複累加
- 若只有學校專案、實習，標記 `is_fresh_graduate: true`

## 特殊情況處理

| 情況 | 處理方式 |
|------|---------|
| 英文履歷 | 正常萃取，欄位值保留英文 |
| 中英混合 | 技能名稱保留原文（如 React、Docker） |
| 沒有明確職稱 | 從最近工作經歷推斷 |
| 履歷格式特殊 | 優先視覺化讀取（rasterize） |
